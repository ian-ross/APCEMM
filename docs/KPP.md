# KPP subsystem

`KPP` contains chemistry code generated by the Kinetic PreProcessor plus APCEMM wrappers for forward and adjoint integration.

## Main code

- `include/KPP/KPP.hpp` - C-compatible public entry points.
- `include/KPP/KPP_Global.h`, `KPP_Parameters.h`, `KPP_Sparse.h` - generated dimensions, globals, sparse structures.
- `src/KPP/KPP_Integrator.cpp`, `KPP_Integrator_ADJ.cpp` - stiff ODE integrators.
- `src/KPP/KPP_Function.cpp`, `KPP_Rates.cpp`, `KPP_Jacobian*.cpp`, `KPP_Hessian*.cpp` - generated chemistry functions and derivatives.
- `src/KPP/KPP_Photol.cpp`, `KPP_HetRates.cpp` - photolysis and heterogeneous-rate updates.
- `src/KPP/UCX.spc`, `UCX.eqn`, `UCX.def` - mechanism source files.

## Public API

Important entry points declared in `KPP.hpp`:

- `INTEGRATE` - forward chemical integration from `TIN` to `TOUT`.
- `INTEGRATE_ADJ` and `KPP_Main_ADJ` - adjoint integration paths.
- `Update_RCONST` - update temperature/pressure/air-density/water-dependent reaction constants.
- `Update_JRates` - compute photolysis rates from solar zenith information.
- `GC_SETHET` - set heterogeneous chemistry rates from aerosol/ice surface properties.
- `ComputeFamilies` - aggregate species families/rates.

## Chemistry role in current plume path

The main executable currently runs only the LAGRID plume model. In that path, KPP is still linked and used where chemistry is needed during EPM/background initialization. Broader plume chemistry and adjoint functionality exist in the codebase but are not the active `Main.cpp` branch.

## Thread-safety considerations

KPP exposes many generated globals: species concentrations, rate constants, photolysis rates, heterogeneous rates, time, and solar-zenith constants. `KPP_Global.h` declares key mutable globals as OpenMP `threadprivate`, but this remains a major architectural constraint:

- new parallel chemistry calls must verify all mutated state is thread-local or protected;
- initialization of threadprivate state must be done per thread as needed;
- C-compatible arrays and generated dimensions must match `KPP_Parameters.h`.

## Physics represented

KPP handles gas-phase and heterogeneous chemical kinetics: species concentration ODEs, reaction rate constants, photolysis, heterogeneous uptake on aerosol/ice, Jacobians/Hessians for stiff integration and adjoints, and family diagnostics.
